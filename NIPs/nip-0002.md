## Summary

The Noven Fork is the largest protocol upgrade in Ethernova history. It activates at mainnet block 480,000 and introduces adaptive gas pricing, native reentrancy protection, 9 precompile contracts, state expiry, Tempo transactions, Frame account abstraction, and anti-MEV fair ordering.

## Motivation

Ethereum's EVM has known limitations that have been discussed for years but never implemented: flat gas pricing that doesn't reward efficient code, no protocol-level reentrancy protection, unbounded state growth, and susceptibility to MEV extraction. Ethernova implements these improvements at Layer 1.

## Specification

### Activation

- **Fork block**: 480,000 (mainnet, chain ID 121525)
- **Required version**: v2.0.0 or higher
- **Enforcement**: Nodes running < v2.0.0 will refuse to start past block 480,000

### Adaptive Gas V2

Post-execution gas adjustment based on trace counters:

| Category | SSTORE count | Adjustment |
|----------|-------------|------------|
| Pure | 0 | up to -25% |
| Light | 0 (reads only) | up to -15% |
| Mixed | 1-4 | 0% |
| Complex | 5+ | up to +10% |

Applied to execution gas only (intrinsic gas is never modified). Uses `computeStablePenaltyScore()` which excludes SLOAD and JUMPI from the penalty calculation to ensure deterministic results regardless of code path.

### Native Precompiles

### Per-EVM Reentrancy Guard

- Blocks self-reentrancy (contract A calling A while already in A's stack)
- Allows cross-contract calls (A -> B -> C)
- Per-EVM instance (no interference between concurrent eth_call/miner/tracer)

### Gas Refund on Revert

- 90% of execution gas refunded on revert
- Anti-DoS: only applies if execution gas < 100,000
- Constant (not runtime-togglable)

### State Expiry

- Contracts with no activity for 900,000 blocks (~115 days) are archived
- EOA wallets are never expired
- Archived state recoverable with Merkle proof

### Tempo Transactions

- Atomic batching (up to 16 calls)
- Fee delegation
- Scheduled execution (up to 500 blocks ahead)
- All-or-nothing execution semantics

### Anti-MEV Fair Ordering

- FIFO transaction ordering by arrival time
- Rate limiting: max 16 pending txs per sender

## Backwards Compatibility

100% backwards compatible. All existing contracts, tools, and wallets work without modification. Block hashes verified identical from block 1 to 445,000 between v1.3.x and v2.0.0.

## Security Considerations

- 5 consensus bugs identified and fixed during devnet testing (v1.1.0 through v1.1.10)
- fastMode permanently disabled (caused different gas computation across nodes)
- GlobalReentrancyGuard replaced with per-EVM guard (prevented cross-instance interference)
- 53 unit tests passing
- Adaptive gas uses only integer math (no floats, no maps, no global mutable state)
